# 🌳 Family Tree - Vue.js Web & Mobile App

A modern, cross-platform family tree application migrated from WordPress to Vue.js with **single codebase** for web and mobile (iOS/Android).

## ✨ Features

### Core Features
- 👥 **Multi-tenant Family Trees** - Each family has private, isolated data
- 🔐 **Role-Based Access Control** - Admin, Editor, Viewer roles
- 👤 **Person Management** - Add family members with photos, bio, dates
- 💑 **Relationship Mapping** - Parent-child, spouse relationships
- 🌲 **Interactive Tree Visualization** - Zoom, pan, search, multiple themes
- 📄 **PDF Export** - 5 templates, multiple sizes, watermark control
- 💰 **Subscription System** - **Family-based plans** (Free → Elite) | GCash, Maya, Cards
- 📧 **Invitation System** - Email invitations and join codes
- 📊 **Activity Logging** - Track all changes
- 🔌 **REST API** - Full API access for integrations

### 🎫 Support Ticket System (NEW!)
- Submit feature suggestions, bug reports, or issues
- Attach screenshots
- **Get rewarded** with 1-15 additional person slots
- Admin panel for ticket management
- Auto-reward system for approved features and critical bugs

### 📱 Mobile Features
- Native iOS and Android apps
- Camera integration for photos
- Native sharing
- Platform detection
- Same codebase as web app!

## 🛠 Technology Stack

### Frontend (Web + Mobile - Single Codebase)
- **Vue.js 3** + TypeScript + Vite
- **Capacitor** - Converts Vue.js to native iOS/Android apps
- **Pinia** - State management
- **Vue Router** - Navigation
- **Tailwind CSS** - Styling
- **PrimeVue** - UI components
- **Axios** - API client

### Backend
- **FastAPI** - Modern Python API framework
- **Python 3.10+**
- **Pydantic** - Data validation
- **JWT** - Authentication
- **ReportLab** - PDF generation

### Database
- **MongoDB** (recommended) or **MySQL** (XAMPP compatible)
- Flexible schema for hierarchical data

### DevOps
- **Docker** + Docker Compose
- **Supervisor** - Process management
- Auto-generated API docs (Swagger/OpenAPI)

## 🚀 Quick Start

### Prerequisites
- Python 3.10+
- Node.js 18+
- MongoDB or MySQL (XAMPP)
- (Optional) Xcode for iOS, Android Studio for Android

### 1. Clone Repository
```bash
git clone https://github.com/YOUR-USERNAME/family-tree-vue-app.git
cd family-tree-vue-app
```

### 2. Backend Setup
```bash
cd backend
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env
# Edit .env with your database connection

# Run backend
uvicorn app.main:app --reload --port 8001
```

### 3. Frontend Setup
```bash
cd frontend
yarn install
cp .env.example .env
# Edit .env with API URL

# Run frontend
yarn dev
```

### 4. Access Application
- **Frontend:** http://localhost:5173
- **Backend API:** http://localhost:8001
- **API Docs:** http://localhost:8001/docs

## 📱 Mobile Development

### Build for iOS
```bash
cd frontend
yarn build
npx cap add ios
npx cap sync ios
npx cap open ios
# Run in Xcode
```

### Build for Android
```bash
cd frontend
yarn build
npx cap add android
npx cap sync android
npx cap open android
# Run in Android Studio
```

## 📊 API Endpoints

### Authentication
- `POST /api/auth/register` - Register
- `POST /api/auth/login` - Login
- `POST /api/auth/refresh` - Refresh token

### Families
- `GET /api/families` - List families
- `POST /api/families` - Create family
- `GET /api/families/{id}` - Get family
- `PUT /api/families/{id}` - Update family
- `DELETE /api/families/{id}` - Delete family

### Persons
- `GET /api/families/{family_id}/persons` - List persons
- `POST /api/families/{family_id}/persons` - Create person
- `GET /api/persons/{id}` - Get person
- `PUT /api/persons/{id}` - Update person
- `DELETE /api/persons/{id}` - Delete person

### Support Tickets
- `POST /api/tickets` - Submit ticket
- `GET /api/tickets/my-tickets` - My tickets
- `POST /api/tickets/upload-screenshot` - Upload screenshot
- `GET /api/tickets/admin/all` - All tickets (admin)
- `POST /api/tickets/admin/{id}/reward` - Reward user (admin)

[View complete API documentation](http://localhost:8001/docs)
