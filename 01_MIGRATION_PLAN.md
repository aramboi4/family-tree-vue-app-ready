# Family Tree WordPress Plugin вҶ’ Vue.js + Mobile App Migration Plan

## рҹҺҜ Project Overview

Migrating WordPress Family Tree SaaS Plugin to a modern, cross-platform application using **single codebase** for both web and mobile.

## рҹӣ  Technology Stack

### Frontend (Web + Mobile - Single Codebase)
- **Framework:** Vue.js 3 + TypeScript
- **Build Tool:** Vite
- **Mobile:** Capacitor (converts Vue.js to native iOS/Android)
- **State Management:** Pinia
- **UI Framework:** Tailwind CSS + PrimeVue
- **Tree Visualization:** Custom Canvas + D3.js
- **Routing:** Vue Router

### Backend
- **Framework:** FastAPI (Python)
- **Database:** MongoDB
- **Authentication:** JWT tokens
- **PDF Generation:** ReportLab
- **Image Upload:** Cloudinary or local storage

### DevOps
- **Containerization:** Docker + Docker Compose
- **API Documentation:** OpenAPI/Swagger (auto-generated by FastAPI)

## рҹ“ұ Single Codebase Strategy

### How It Works:
1. **Develop once in Vue.js** - Build the entire app as a Vue.js SPA
2. **Web deployment** - Deploy directly as a web app
3. **Mobile build** - Use Capacitor to wrap the same Vue.js app into native iOS/Android apps
4. **Platform detection** - Use Capacitor APIs to detect platform and adjust UI/behavior
5. **Native features** - Access camera, storage, etc. via Capacitor plugins

### Code Sharing:
```
frontend/
в”ңв”Җв”Җ src/
в”Ӯ   в”ңв”Җв”Җ components/     # 100% shared
в”Ӯ   в”ңв”Җв”Җ views/          # 100% shared
в”Ӯ   в”ңв”Җв”Җ stores/         # 100% shared
в”Ӯ   в”ңв”Җв”Җ services/       # 100% shared
в”Ӯ   в”ңв”Җв”Җ composables/    # Platform-specific logic here
в”Ӯ   в””в”Җв”Җ utils/          # 100% shared
```

### Platform-Specific Code Example:
```typescript
// composables/usePlatform.ts
import { Capacitor } from '@capacitor/core'

export const usePlatform = () => {
  const isWeb = Capacitor.getPlatform() === 'web'
  const isMobile = Capacitor.isNativePlatform()
  const isIOS = Capacitor.getPlatform() === 'ios'
  const isAndroid = Capacitor.getPlatform() === 'android'
  
  return { isWeb, isMobile, isIOS, isAndroid }
}
```

## рҹҡҖ Development Workflow

### 1. Initial Setup
```bash
# Clone/create project
git clone <repo>
cd family-tree-app

# Backend setup
cd backend
python -m venv venv
source venv/bin/activate  # or venv\Scripts\activate on Windows
pip install -r requirements.txt
cp .env.example .env
# Edit .env with MongoDB connection string

# Frontend setup
cd ../frontend
yarn install
cp .env.example .env
# Edit .env with API URL
```

### 2. Development (Web)
```bash
# Terminal 1: Backend
cd backend
uvicorn app.main:app --reload --port 8001

# Terminal 2: Frontend
cd frontend
yarn dev
```

### 3. Mobile Development
```bash
# Build web app
cd frontend
yarn build

# Add platforms (first time only)
npx cap add ios
npx cap add android

# Sync web build to mobile
npx cap sync

# Open in native IDE
npx cap open ios      # Opens Xcode
npx cap open android  # Opens Android Studio

# Run on device/emulator
# Use Xcode or Android Studio to run
```

### 4. Testing
```bash
# Backend tests
cd backend
pytest

# Frontend tests
cd frontend
yarn test
```

## рҹ“Ҡ API Endpoints (Backend)

### Authentication
- POST `/api/auth/register` - Register new user
- POST `/api/auth/login` - Login
- POST `/api/auth/refresh` - Refresh token
- POST `/api/auth/forgot-password` - Request password reset
- POST `/api/auth/reset-password` - Reset password

### Families
- GET `/api/families` - List user's families
- POST `/api/families` - Create family
- GET `/api/families/{id}` - Get family details
- PUT `/api/families/{id}` - Update family
- DELETE `/api/families/{id}` - Delete family
- GET `/api/families/{id}/members` - Get family members
- POST `/api/families/{id}/invite` - Invite member

### Persons
- GET `/api/families/{family_id}/persons` - List persons
- POST `/api/families/{family_id}/persons` - Create person
- GET `/api/persons/{id}` - Get person details
- PUT `/api/persons/{id}` - Update person
- DELETE `/api/persons/{id}` - Delete person
- POST `/api/persons/{id}/photo` - Upload photo

### Relationships
- GET `/api/families/{family_id}/relationships` - List relationships
- POST `/api/relationships` - Create relationship
- DELETE `/api/relationships/{id}` - Delete relationship

### Subscriptions
- GET `/api/subscriptions/plans` - List plans
- GET `/api/subscriptions/current` - Get current subscription
- POST `/api/subscriptions/subscribe` - Subscribe to plan
- PUT `/api/subscriptions/cancel` - Cancel subscription

### PDF Export
- POST `/api/pdf/export` - Generate PDF
- GET `/api/pdf/{id}` - Download PDF

### Activity
- GET `/api/families/{family_id}/activity` - Get activity log
